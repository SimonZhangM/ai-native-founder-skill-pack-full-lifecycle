# Launch and Scale Operations Reference

## Launch readiness

Launch is about proving the business deserves to grow.

A Launch-stage startup must harden:

- Product infrastructure.
- Security and compliance.
- Technical debt.
- Product management process.
- Feedback loops.
- Growth channels.
- Founder bottleneck removal.

## Technical debt audit

Classify findings as:

1. Fix before next release.
2. Fix in next sprint.
3. Acceptable debt for current stage.
4. Needs human/security/compliance review.

Inspect:

- Brittle architecture.
- Inconsistent patterns from AI-generated sessions.
- Test coverage gaps.
- Missing architecture decisions.
- Unsafe auth/data handling.
- Overcomplicated features from scope creep.

## Founder bottleneck audit

Inventory:

- Decisions only the founder makes.
- Tasks only the founder remembers.
- Customer questions only the founder can answer.
- Support workflows only the founder triages.
- Reports only the founder compiles.
- Product changes only the founder can scope.

Classify each as:

- Founder-only judgment.
- Delegate to human.
- Automate with AI.
- Convert to documented process.
- Stop doing.

## Scale readiness

Scale requires:

- Enterprise-grade support.
- Documentation and SLAs.
- Incident response.
- Logging, monitoring, and observability.
- Governance and compliance posture.
- GTM system beyond founder-led selling.
- Workflow lock-in.
- Accumulated domain knowledge.
- Data flywheel.

## GTM system components

- Market segmentation.
- Messaging architecture.
- Sales playbook.
- Customer success playbook.
- Analyst/investor narrative.
- CRM hygiene.
- Pipeline reporting.
- Content and outbound cadence.
- Technical demos, sandboxes, API docs, integration guides.

## Moat audit

A defensible AI-native company should be able to explain:

1. What domain knowledge has been encoded into workflows, tests, prompts, integrations, or product logic?
2. What user behavior data creates a compounding feedback loop?
3. What workflows or integrations make the product costly to leave?
4. Which edge cases would a generalist competitor get wrong?
5. Why would customers stay if a well-funded incumbent copied the visible product?
